# Day 16 연습문제 - Next.js 시작, App Router

## 연습문제 1: 포트폴리오 첫 페이지

### 목표
Next.js App Router를 사용하여 자기소개 포트폴리오 페이지를 만드세요.

### 요구사항
- `layout.tsx`에 사이트 제목(메타데이터)을 설정하세요.
- `page.tsx`에 다음 내용을 표시하세요:
  - 이름 (h1 태그)
  - 한 줄 소개 (p 태그)
  - 기술 스택 목록 (ul/li 태그, 최소 5개)
- 기본적인 스타일링을 적용하세요 (인라인 스타일 또는 CSS 모듈).

### 기대 결과
```
[브라우저 탭 제목: "나의 포트폴리오"]

홍길동
프론트엔드 개발자를 꿈꾸는 학생입니다.

기술 스택:
- JavaScript
- TypeScript
- React
- Next.js
- CSS
```

### 힌트
- `layout.tsx`는 모든 페이지에 공통으로 적용되는 껍데기입니다.
- `metadata` 객체를 export하면 `<title>` 태그가 자동 설정됩니다.
- `page.tsx`는 해당 경로(`/`)에서 보여줄 실제 콘텐츠입니다.

## 연습문제 2: 'use client' 이해하기

### 목표
서버 컴포넌트와 클라이언트 컴포넌트의 차이를 이해하고, 카운터를 만들어보세요.

### 요구사항
- 버튼을 클릭하면 숫자가 1씩 증가하는 카운터를 만드세요.
- `useState`를 사용하세요.
- 파일 상단에 `'use client'`를 넣어야 하는 이유를 주석으로 설명하세요.
- "왜 page.tsx에 바로 useState를 쓸 수 없는지" 주석으로 적으세요.

### 기대 결과
```
현재 카운트: 0
[+1 증가] 버튼

버튼 클릭 시 -> 현재 카운트: 1, 2, 3, ...
```

### 힌트
- Next.js App Router에서 모든 컴포넌트는 기본적으로 서버 컴포넌트입니다.
- `useState`, `useEffect`, 이벤트 핸들러(`onClick`) 등은 클라이언트에서만 동작합니다.
- 클라이언트 기능이 필요한 컴포넌트에만 `'use client'`를 선언하세요.

## 연습문제 3: 레이아웃 활용

### 목표
`layout.tsx`에 헤더와 푸터를 넣어 공통 레이아웃을 만드세요.

### 요구사항
- `layout.tsx`에 다음을 구현하세요:
  - 상단 헤더: 사이트명 "My Portfolio" 표시
  - 하단 푸터: "© 2025 My Portfolio. All rights reserved." 표시
  - 헤더와 푸터 사이에 `{children}`이 렌더링되어야 합니다.
- `page.tsx`에 간단한 환영 메시지를 넣으세요.
- `children`이 무엇인지, 왜 필요한지 주석으로 설명하세요.

### 기대 결과
```
=========================
  My Portfolio (헤더)
=========================

  환영합니다!
  이곳은 나의 포트폴리오입니다.

=========================
  © 2025 My Portfolio (푸터)
=========================
```

### 힌트
- `layout.tsx`의 `children` prop은 현재 라우트의 `page.tsx` 내용입니다.
- 레이아웃은 페이지가 바뀌어도 다시 렌더링되지 않습니다 (상태 유지).
- `<header>`, `<main>`, `<footer>` 시맨틱 태그를 사용하면 좋습니다.
