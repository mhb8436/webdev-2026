# Day 17 연습문제 - 라우팅, 레이아웃

## 연습문제 1: 미니 쇼핑몰 라우팅

### 목표
폴더 기반 라우팅으로 쇼핑몰의 기본 페이지 구조를 만드세요.

### 요구사항
- 다음 4개 페이지를 만드세요:
  - `/` (홈) - 환영 메시지, 추천 상품 안내
  - `/products` (상품 목록) - 상품 3~5개를 목록으로 표시
  - `/products/[id]` (상품 상세) - URL의 id를 받아 해당 상품 정보 표시
  - `/cart` (장바구니) - "장바구니가 비어있습니다" 메시지 표시
- 모든 페이지에 공통 네비게이션 바를 만드세요 (Link 컴포넌트 사용).
- 상품 목록에서 각 상품을 클릭하면 상세 페이지로 이동하세요.

### 폴더 구조
```
app/
├── layout.tsx          (공통 네비게이션)
├── page.tsx            (홈)
├── products/
│   ├── page.tsx        (상품 목록)
│   └── [id]/
│       └── page.tsx    (상품 상세)
└── cart/
    └── page.tsx        (장바구니)
```

### 기대 결과
```
[홈] [상품목록] [장바구니]     <- 네비게이션 (모든 페이지 공통)

--- /products 페이지 ---
상품 목록
- 무선 키보드 (29,000원)  [상세보기 ->]
- 무선 마우스 (19,000원)  [상세보기 ->]
- USB 허브 (15,000원)     [상세보기 ->]

--- /products/1 페이지 ---
무선 키보드
가격: 29,000원
설명: 조용한 타건감의 블루투스 키보드
[목록으로 돌아가기]
```

### 힌트
- `next/link`의 `Link` 컴포넌트로 클라이언트 사이드 네비게이션을 구현합니다.
- 동적 라우트 `[id]`는 `params` prop으로 값을 받을 수 있습니다.
- `<a>` 태그 대신 `Link`를 쓰면 페이지 전체가 새로고침되지 않습니다.

## 연습문제 2: 중첩 레이아웃

### 목표
블로그 섹션에만 적용되는 중첩 레이아웃을 만드세요.

### 요구사항
- 루트 `layout.tsx`: 사이트 전체 헤더 (사이트명 + 네비게이션)
- `/blog/layout.tsx`: 블로그 전용 사이드바 (카테고리 목록)
- `/blog/page.tsx`: 블로그 글 목록
- `/blog/[slug]/page.tsx`: 블로그 글 상세
- 블로그 페이지에서는 헤더 + 사이드바 + 본문이 함께 보여야 합니다.
- 다른 페이지(예: `/about`)에서는 사이드바가 보이지 않아야 합니다.

### 폴더 구조
```
app/
├── layout.tsx          (루트 레이아웃 - 헤더)
├── page.tsx            (홈)
├── about/
│   └── page.tsx        (소개 - 사이드바 없음)
└── blog/
    ├── layout.tsx      (블로그 레이아웃 - 사이드바)
    ├── page.tsx        (글 목록)
    └── [slug]/
        └── page.tsx    (글 상세)
```

### 기대 결과
```
=== /blog 페이지 ===
[헤더: 사이트명 + 네비게이션]        <- 루트 레이아웃
┌──────────┬────────────────────┐
│ 카테고리  │  블로그 글 목록     │   <- 블로그 레이아웃
│ - 기술   │  - React 입문       │
│ - 일상   │  - Next.js 시작     │
│ - 여행   │  - CSS 팁           │
└──────────┴────────────────────┘

=== /about 페이지 ===
[헤더: 사이트명 + 네비게이션]        <- 루트 레이아웃
(사이드바 없음)
소개 페이지 내용...
```

### 힌트
- 각 폴더에 `layout.tsx`를 만들면 해당 경로 하위에만 적용됩니다.
- 레이아웃은 중첩됩니다: 루트 레이아웃 > 블로그 레이아웃 > 페이지.
- CSS Flexbox로 사이드바와 본문을 나란히 배치하세요.

## 연습문제 3: 활성 링크 표시

### 목표
현재 페이지에 해당하는 네비게이션 링크를 시각적으로 강조하세요.

### 요구사항
- 네비게이션에 홈, 소개, 블로그, 연락처 링크를 만드세요.
- `usePathname()` 훅으로 현재 경로를 가져오세요.
- 현재 페이지의 링크에 다른 스타일을 적용하세요:
  - 활성: 굵은 글씨 + 밑줄 + 파란색
  - 비활성: 일반 글씨 + 회색
- 네비게이션 컴포넌트는 클라이언트 컴포넌트로 만들어야 합니다.

### 기대 결과
```
/ 경로일 때:       [홈(활성)]  소개  블로그  연락처
/about 경로일 때:   홈  [소개(활성)]  블로그  연락처
/blog 경로일 때:    홈  소개  [블로그(활성)]  연락처
```

### 힌트
- `usePathname()`은 `next/navigation`에서 import합니다.
- 이 훅은 클라이언트에서만 동작하므로 `'use client'`가 필요합니다.
- 조건부 스타일: `pathname === href ? activeStyle : defaultStyle`
- `pathname.startsWith('/blog')`처럼 하위 경로도 활성화할 수 있습니다.
