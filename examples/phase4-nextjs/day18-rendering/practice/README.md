# Day 18 연습문제 - Server/Client 컴포넌트

## 연습문제 1: 뉴스 피드

### 목표
서버 컴포넌트와 클라이언트 컴포넌트를 적절히 분리하여 뉴스 피드를 만드세요.

### 요구사항
- `page.tsx` (서버 컴포넌트)에서 뉴스 데이터를 하드코딩 배열로 준비하세요.
- 뉴스 목록을 렌더링하되, 각 뉴스 항목에 "좋아요" 버튼을 포함하세요.
- "좋아요" 버튼은 클릭하면 카운트가 증가해야 합니다.
- "좋아요" 기능은 별도의 클라이언트 컴포넌트 `LikeButton.tsx`로 분리하세요.
- 서버/클라이언트 컴포넌트 구분의 이유를 주석으로 설명하세요.

### 뉴스 데이터 예시
```typescript
const newsItems = [
  { id: 1, title: "Next.js 15 출시", summary: "새로운 기능들이 추가되었습니다.", author: "김개발" },
  { id: 2, title: "React 19 업데이트", summary: "서버 컴포넌트가 안정화되었습니다.", author: "이리액트" },
  { id: 3, title: "TypeScript 6.0 발표", summary: "타입 시스템이 더욱 강력해졌습니다.", author: "박타입" },
];
```

### 기대 결과
```
최신 뉴스

┌─────────────────────────────────┐
│ Next.js 15 출시                  │
│ 새로운 기능들이 추가되었습니다.    │
│ 작성자: 김개발                   │
│ [좋아요 0] <- 클릭하면 1, 2, ... │
└─────────────────────────────────┘

┌─────────────────────────────────┐
│ React 19 업데이트                │
│ ...                              │
└─────────────────────────────────┘
```

### 힌트
- 서버 컴포넌트: 데이터를 가져오고 HTML을 렌더링하는 데 적합 (DB 접근, API 호출 등)
- 클라이언트 컴포넌트: 사용자 상호작용(클릭, 입력)이 필요한 부분에 사용
- 서버 컴포넌트에서 클라이언트 컴포넌트를 import하여 사용할 수 있습니다.

## 연습문제 2: 서버/클라이언트 판별

### 목표
5개 컴포넌트를 만들고 각각 서버/클라이언트 중 적절한 것을 선택하세요.

### 요구사항
다음 5개 컴포넌트를 만드세요. 각각 서버/클라이언트 어느 쪽이 적합한지 판단하고, 이유를 주석으로 적으세요.

1. **Header** - 사이트 로고와 제목 표시
2. **Footer** - 저작권 정보와 링크 표시
3. **SearchBar** - 사용자가 텍스트를 입력하여 검색하는 바
4. **ArticleList** - 기사 목록을 서버에서 가져와 표시
5. **LikeButton** - 클릭하면 좋아요 수가 증가하는 버튼

### 판단 기준
| 기준 | 서버 컴포넌트 | 클라이언트 컴포넌트 |
|------|-------------|-------------------|
| 데이터 가져오기 | O | X (가능하지만 비효율적) |
| useState/useEffect | X | O |
| onClick 등 이벤트 | X | O |
| 정적 UI | O (최적) | O (불필요한 JS 전송) |
| 브라우저 API | X | O |

### 기대 결과
```
[Header]      - 서버 컴포넌트 (정적 UI, 상호작용 없음)
[SearchBar]   - 클라이언트 컴포넌트 (입력 상태, onChange 이벤트)
[ArticleList] - 서버 컴포넌트 (데이터 표시, 상호작용 없음)
[LikeButton]  - 클라이언트 컴포넌트 (클릭 이벤트, 상태 관리)
[Footer]      - 서버 컴포넌트 (정적 UI)
```

### 힌트
- "이 컴포넌트가 useState나 이벤트 핸들러를 사용하는가?"가 핵심 질문입니다.
- 사용하지 않는다면 서버 컴포넌트(기본)로 두는 것이 좋습니다.
- 서버 컴포넌트는 JavaScript 번들에 포함되지 않아 성능에 유리합니다.

## 연습문제 3: 데이터 전달 패턴

### 목표
서버 컴포넌트에서 데이터를 준비하고, 클라이언트 컴포넌트에 props로 전달하는 패턴을 연습하세요.

### 요구사항
- `page.tsx` (서버 컴포넌트)에서 월별 판매 통계를 계산하세요.
- 계산된 통계를 `StatsChart.tsx` (클라이언트 컴포넌트)에 props로 전달하세요.
- 클라이언트 컴포넌트에서는 전달받은 데이터로 간단한 막대 차트를 표시하세요.
- 막대 차트는 div의 width를 % 로 조절하여 표현하세요 (라이브러리 불필요).

### 통계 데이터 예시
```typescript
// 서버에서 계산 (page.tsx)
const monthlySales = [
  { month: "1월", amount: 120 },
  { month: "2월", amount: 98 },
  { month: "3월", amount: 150 },
  { month: "4월", amount: 200 },
  { month: "5월", amount: 175 },
  { month: "6월", amount: 220 },
];

const total = monthlySales.reduce((sum, s) => sum + s.amount, 0);
const average = Math.round(total / monthlySales.length);
const maxMonth = monthlySales.reduce((max, s) => s.amount > max.amount ? s : max);
```

### 힌트
- 서버에서 계산을 수행하면 클라이언트에 불필요한 연산을 보내지 않습니다.
- props로 전달하는 데이터는 직렬화 가능해야 합니다 (함수, Date 등은 불가).
- 클라이언트 컴포넌트에서 호버(onMouseEnter) 등 인터랙션을 추가하세요.
