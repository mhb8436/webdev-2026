# Day 19 연습문제 - metadata, loading, error, not-found

## 연습문제 1: 동적 메타데이터

### 목표
블로그 글 상세 페이지에 `generateMetadata`로 동적 메타데이터를 설정하세요.

### 요구사항
- `/blog/[slug]` 경로에 블로그 상세 페이지를 만드세요.
- `generateMetadata` 함수를 사용하여 글 제목을 페이지 `<title>`로 설정하세요.
- 글 데이터는 하드코딩 배열에서 slug로 검색하세요.
- 존재하지 않는 slug로 접근하면 `notFound()`를 호출하세요.
- `not-found.tsx` 파일도 만들어 커스텀 404 페이지를 구현하세요.

### 블로그 데이터 예시
```typescript
const posts = [
  { slug: "nextjs-intro", title: "Next.js 입문 가이드", content: "Next.js는 React 기반의 풀스택 프레임워크입니다..." },
  { slug: "react-hooks", title: "React Hooks 완벽 정리", content: "useState, useEffect, useRef 등..." },
  { slug: "typescript-tips", title: "TypeScript 실전 팁", content: "타입 가드, 제네릭 활용법..." },
];
```

### 기대 결과
```
--- /blog/nextjs-intro 접속 시 ---
[브라우저 탭: "Next.js 입문 가이드 | 내 블로그"]

Next.js 입문 가이드
Next.js는 React 기반의 풀스택 프레임워크입니다...

--- /blog/없는슬러그 접속 시 ---
[브라우저 탭: "페이지를 찾을 수 없습니다"]

404
요청하신 블로그 글을 찾을 수 없습니다.
[블로그 목록으로 돌아가기]
```

### 힌트
- `generateMetadata`는 `async` 함수로, `params`에서 `slug`를 받습니다.
- `notFound()`는 `next/navigation`에서 import합니다.
- `not-found.tsx`는 `notFound()` 호출 시 렌더링되는 커스텀 페이지입니다.
- `generateMetadata`와 페이지 컴포넌트 둘 다 같은 파일에 export할 수 있습니다.

## 연습문제 2: 스켈레톤 로딩

### 목표
실제 콘텐츠와 유사한 형태의 스켈레톤 UI를 만드세요.

### 요구사항
- `loading.tsx`에 스켈레톤 UI를 구현하세요.
- 실제 페이지 레이아웃과 비슷한 형태의 플레이스홀더를 만드세요:
  - 제목 위치: 넓은 회색 막대
  - 본문 위치: 여러 줄의 좁은 회색 막대
  - 이미지 위치: 회색 사각형
- CSS 애니메이션으로 깜빡이는(pulse) 효과를 추가하세요.
- 의도적으로 로딩을 확인할 수 있도록 데이터 지연을 시뮬레이션하세요.

### 기대 결과
```
로딩 중 (스켈레톤):
┌────────────────────────────┐
│ ████████████████           │  <- 제목 스켈레톤 (깜빡임)
│                            │
│ ████████████████████████   │  <- 본문 줄 1 (깜빡임)
│ █████████████████████      │  <- 본문 줄 2 (깜빡임)
│ ████████████████           │  <- 본문 줄 3 (깜빡임)
│                            │
│ ┌──────────────────┐       │
│ │   이미지 자리     │       │  <- 이미지 스켈레톤 (깜빡임)
│ └──────────────────┘       │
└────────────────────────────┘

로딩 완료 후: 실제 콘텐츠 표시
```

### 힌트
- `@keyframes pulse` 애니메이션: opacity를 0.6 <-> 1 사이로 반복
- `loading.tsx`는 같은 폴더 또는 상위 폴더에 위치하면 자동 적용됩니다.
- 인라인 스타일에서 animation을 사용하려면 별도의 `<style>` 태그가 필요합니다.
- 로딩 테스트: 페이지 컴포넌트에서 `await new Promise(r => setTimeout(r, 2000))`로 지연

## 연습문제 3: 에러 복구

### 목표
에러 바운더리(`error.tsx`)를 구현하고, 에러 복구 기능을 만드세요.

### 요구사항
- `error.tsx`에 사용자 친화적인 에러 UI를 만드세요:
  - 에러 메시지를 보기 좋게 표시
  - "다시 시도" 버튼으로 `reset()` 호출
  - 홈으로 돌아가는 링크도 제공
- 의도적으로 에러를 발생시키는 페이지를 만들어 테스트하세요.
  - 예: 랜덤으로 에러가 발생하는 페이지
  - 또는 특정 조건에서 throw하는 페이지

### 기대 결과
```
--- 정상 동작 시 ---
랜덤 명언
"오늘도 좋은 하루 되세요!"

--- 에러 발생 시 ---
┌────────────────────────────────────┐
│                                    │
│     문제가 발생했습니다!             │
│                                    │
│     죄송합니다.                     │
│     페이지를 불러오는 중             │
│     오류가 발생했습니다.             │
│                                    │
│     [다시 시도]  [홈으로 돌아가기]    │
│                                    │
└────────────────────────────────────┘
```

### 힌트
- `error.tsx`는 반드시 클라이언트 컴포넌트(`'use client'`)여야 합니다.
- props로 `error`(Error 객체)와 `reset`(함수)을 받습니다.
- `reset()`은 에러가 발생한 컴포넌트를 다시 렌더링합니다.
- 개발 모드에서는 `error.message`로 상세 에러 메시지를 확인할 수 있습니다.
- 에러 테스트: `if (Math.random() > 0.5) throw new Error("랜덤 에러!");`
