# 할일 관리 앱 데이터베이스 요구사항

## 기능 요구사항

1. 여러 사용자가 각자의 할일을 관리할 수 있다
2. 할일은 카테고리로 분류할 수 있다
3. 하나의 카테고리에 여러 할일이 속할 수 있다
4. 각 사용자는 자신만의 카테고리를 만들 수 있다

## 필요한 테이블

### users (사용자 정보)
- 사용자 고유 식별자
- 사용자 이름
- 이메일 주소
- 가입일

### categories (카테고리 정보)
- 카테고리 고유 식별자
- 카테고리 이름
- 어떤 사용자의 카테고리인지 (소유자)
- 생성일

### todos (할일 정보)
- 할일 고유 식별자
- 할일 제목
- 상세 설명
- 완료 여부
- 우선순위
- 어떤 사용자의 할일인지 (소유자)
- 어떤 카테고리에 속하는지
- 생성일
- 수정일

## 관계 설계

```
users 1 ──── N categories   (한 사용자는 여러 카테고리를 가진다)
users 1 ──── N todos        (한 사용자는 여러 할일을 가진다)
categories 1 ── N todos     (한 카테고리에 여러 할일이 속한다)
```
