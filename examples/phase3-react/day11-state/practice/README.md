# Day 11 연습문제 - useState, 이벤트

## 학습 목표
- useState로 컴포넌트의 상태를 관리할 수 있다
- 이벤트 핸들러를 통해 사용자 인터랙션을 처리할 수 있다
- 여러 개의 state를 조합하여 복잡한 UI를 만들 수 있다

## 문제 1: 장바구니

상품 목록에서 장바구니에 상품을 추가/삭제하고, 총 금액을 계산하는 기능을 만드세요.

### 요구사항
- 상품 목록(최소 5개)을 배열로 준비하세요 (이름, 가격 포함)
- 각 상품 옆에 "담기" 버튼을 표시하세요
- "담기" 버튼을 누르면 장바구니 배열에 해당 상품이 추가됩니다
- 장바구니에서 "삭제" 버튼을 누르면 해당 상품이 제거됩니다
- 장바구니의 총 금액을 실시간으로 계산하여 표시하세요
- 장바구니가 비어있으면 "장바구니가 비어있습니다" 메시지를 표시하세요
- `useState`로 장바구니 상태를 관리하세요

### 예시 동작
```
[상품 목록]
노트북 1,200,000원 [담기]
마우스 35,000원 [담기]
키보드 89,000원 [담기]

[장바구니] (2개)
노트북 1,200,000원 [삭제]
키보드 89,000원 [삭제]

총 금액: 1,289,000원
```

## 문제 2: 가위바위보 게임

컴퓨터와 가위바위보 대결을 하는 게임을 만드세요.

### 요구사항
- 가위, 바위, 보 버튼 3개를 만드세요
- 버튼을 클릭하면 컴퓨터가 랜덤으로 하나를 선택합니다
- 내 선택과 컴퓨터 선택을 화면에 표시하세요
- 승/패/무 결과를 판정하여 표시하세요
- 전적(승/패/무 횟수)을 `useState`로 관리하고 표시하세요
- "리셋" 버튼으로 전적을 초기화할 수 있게 하세요

### 승패 판정
- 가위 > 보, 보 > 바위, 바위 > 가위
- 같으면 무승부

## 문제 3: 폼 유효성 검사

실시간으로 유효성을 검사하는 회원가입 폼을 만드세요.

### 요구사항
- 이름, 이메일, 비밀번호 3개 입력 필드를 만드세요
- 각 필드의 유효성을 입력할 때마다 실시간으로 검사하세요:
  - **이름**: 2자 이상
  - **이메일**: @ 포함
  - **비밀번호**: 6자 이상
- 유효하지 않은 필드 아래에 빨간색 에러 메시지를 표시하세요
- 유효한 필드 아래에 초록색 확인 메시지를 표시하세요
- 모든 필드가 유효할 때만 "가입하기" 버튼을 활성화하세요 (disabled 속성 사용)
- 각 필드의 값과 유효성 상태를 `useState`로 관리하세요

### 예시 에러 메시지
- "이름은 2자 이상 입력해주세요"
- "올바른 이메일 형식이 아닙니다"
- "비밀번호는 6자 이상이어야 합니다"

---

## 힌트
- `useState<Product[]>([])` 형태로 배열 상태를 관리합니다
- 배열에 추가: `setItems([...items, newItem])`
- 배열에서 제거: `setItems(items.filter((_, i) => i !== index))`
- 합계 계산: `items.reduce((sum, item) => sum + item.price, 0)`
- 랜덤 선택: `Math.floor(Math.random() * 3)`
